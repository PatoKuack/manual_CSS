CSS Grid

Permite construir layouts a través de dos ejes inline (horizontal) y block (vertical).
No importa la posición e un elemento para construir el layout.
Al igual que flexbox, requiere una relación de padre (grid-container) a hijo (grid-ítems). <émain>

Terminología y elementos.

  • Elementos del DOM
    • grid-container es aquel elemento al que se le aplica en la propiedad display un valor de grid.
      • display: grid; elemento de bloque.
      • display: inline-grid; elemento de línea.
    • grid-ítems é
      • hijos directos del grid.
      • pseudoelementos (::before y ::after).
      • texto (contenido del elemento ítem). é
  • No existen en el DOM, el navegador los crea para posicionar y alinear elementos.
    • grid-tracks
      • filas y columnas.
    • grid-lines
      • se encuentran a lado de los tracks, izquierda y derecha para columnas; arriba y abajo para filas.
    • grid-cells
      • intersección entre una fila y una columna.
    • grid-areas
      • cualquier área rectangular delimitada por cuatro grid-lines.

Propiedades Grid.

Grid-tracks

Se colocan en el elemento padre.

grid-template-columns: #fr #fr; define la cantidad de columnas dentro del grid, se coloca una medida por columna, esa medida definirá el ancho de la columna. La altura de la columna se extenderá hasta el límite inferior del elemento. grid-template-rows: #fr #fr; define la cantidad de filas dentro del grid, se coloca una medida por fila, esa medida definirá la altura de la fila. El largo de la fila se extenderá hasta el límite lateral derecho del elemento. nota1:
La medida en fr distribuye mejor las áreas y sus separaciones sin que se desborden y se pueden combinar los valores (#em #fr #em...)
nota2:
Al colocar una medición en porcentaje a las imágenes dentro de un grid, este porcentaje tomará de referencia el espacio asignado a ese track o área, no al tamaño del navegador.

Valores

min-content especifica que las columnas o filas midan lo mínimo posible tomando como referencia el elemento (palabra o imagen) más grande del contenido, por ejemplo: en una columna, si el contenido contiene 3 palabras, se colocan saltos de línea para que la columna sea lo más angosta posible; ejemplo de uso: grid-template-columns: min-content min content; max-content especifica que las columnas o filas midan lo máximo de contenido posible haciendo que el contenido se acomode para que ocupe todo el espacio posible en horizontal (columna) o vertical (fila). fit-content(#px) función que recibe un parámetro, especifica que las columnas o filas se adapte un mínimo de "min-content" y como máximo el parámetro especificado, ejemplo: grid-template-columns: fit-content(400px);. minmax(#px, #px) coloca lo mínimo y lo máximo en una instrucción, se pueden colocar valores combinados, ejemplo: grid-template-columns: minmax(250px, 400px) 1fr. repeat( N, #fr ); repite N veces una función, maneja dos parámetros, el primero (N) es la cantidad de veces que se repetirá la función para crear los tracks (columnas o filas dependiendo del grid-template que se use) y el segundo es la función que se repetirá. repeat(auto-fit, #px) se utiliza con repeat; verifica si existe espacio y contenido disponible, si se dispone de estos dos factores, anexa otra columna o fila (del tamaño especificado en el segundo valor del repeat), ejemplo: grid-template-columns: repeat(auto-fit, 360px); repeat(auto-fill, #px) se utiliza con repeat; verifica si existe espacio disponible, si lo hay, anexa otra división en columna o fila (del tamaño especificado en el segundo valor del repeat), es necesario que haya contenido para que se visualice la columna o fila agregada, sin embargo no es necesario que haya contenido para crear la columna o fila.
Se puede visualizar mejor con el inspector del navegador web (clic derecho, inspeccionar). Se pueden combinar los valores auto-fill y minmax para obtener un sistema de columnas flexible, ejemplo: grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)); /* agrega columnas dependiendo el espacio disponible, cada columna tendrá como mínimo 200 pixeles y como máximo calcula todo lo que se pueda abarcar. */

Grid-lines

Se coloca en el elemento padre.

grid-column-gap: #em; nos permite agregar separaciones entre columnas (solo se coloca una medición). grid-row-gap: #em; nos permite agregar separaciones entre filas (solo se coloca una medición). grid-gap: [separación para columna] [separación para fila]; es un shorthand para estas propiedades, ejemplo: grid-gap: 0 2em; grid-gap: [separación para columna y fila]; es otro shorthand que permite colocar una medida que defina los tipos de separaciones.

Grid-cells

Se coloca en el elemento hijo.

grid-row-start: #; indica en que grid line de las filas comenzará la colocación del elemento hijo del elemento con grid. grid-row-end: #; indica en que grid line de las filas terminará la colocación del elemento hijo del elemento con grid. grid-column-start: #; indica en que grid line de las columnas comenzará la colocación del elemento hijo del elemento con grid. grid-column-end: #; indica en que grid line de las columnas terminará la colocación del elemento hijo del elemento con grid. grid-row: [start] / [end]; o grid-column: [start] / [end]; es un shorthand, indica en que grid line del renglón o columna empieza y en cual termina el posicionamiento del elemento, ejemplo: grid-row: 1 / 0; grid-row: [start]; o grid-column: [start]; es un shorthand que permite colocar solo un valor que defina únicamente el grid start, ejemplo: grid-column: 3; grid-column: [start] / span #; span solamente se coloca en la posición del end e indica la cantidad de columnas (o filas) que se ocuparán a partir del grid line donde comienza, ejemplo: grid-gap: 1 / span 2; grid-row: span #; permite colocar solo un valor que defina la cantidad de filas o columnas que se van a ocupar, ejemplo: grid-column: span 1; grid-area: [grid-row-start] / [grid-column-start] / [grid-row-end] / [grid-column-end]; es un shorthand que indica el inicio y final de las filas y columnas.

Grid-areas

Cada grid-area es ocupada por un elemento hijo del elemento al que se le aplica el grid.
Se puede definir las áreas e indicarle a los ítems en qué área debe aparecer, permitiendo definir áreas dentro de grid.
Primero se especifican las áreas de los elementos hijos con el comando grid-area:NAME;
Después se especifican las posiciones de las áreas con el comando grid-template-areas: 'NAME2 NAME2 NAME3' 'NAME2 NAME2 NAME1'; cada área declarada (NAME) representa una columna y cada bloque de comillas simples representa una fila.
En el ejemplo se colocaron 3 elementos, el NAME2 ocupa las dos primeras columnas de las dos filas, el NAME3 ocupa la última columna de la primera fila y el NAME1 ocupa la última columna de la segunda fila.

Lo siguiente se coloca en el elemento padre (cada área debe ocupar un rectángulo y no pueden haber dos áreas (rectángulos) hechos con el mismo elemento hijo).

grid-template-areas:
'leaf1 leaf1 sky1 sky1 sky1 sky1 sky1 sky1 leaf2 leaf2'
'leaf1 leaf1 sky1 sky1 sky1 sky1 sky1 sky1 leaf2 leaf2'
'tree1 sky2 sky2 sky2 sun1 sun1 sky3 sky3 sky3 tree2'
'tree1 sky2 sky2 sky2 sun1 sun1 sky3 sky3 sky3 tree2'
'grnd grnd grnd grnd grnd grnd grnd grnd grnd grnd'
'grnd grnd grnd grnd grnd grnd grnd grnd grnd grnd'
'grnd grnd grnd grnd grnd grnd grnd grnd grnd grnd'
;

Lo siguiente se coloca un el elemento hijo (cada nombre de área va en un elemento descendiente del elemento padre).

grid-area: left1;

justify-self propiedad que alinea un ítem especiéico en el eje X, se coloca en el hijo del contenedor grid.

nota:
Valores de justify y de align: [start | end | center]

Alineaciones de tracks

justify-content propiedad que alinea los tracks en el eje X dentro del contenedor, se coloca en el contenedor padre grid.
align-content propiedad que alinea los tracks en el eje Y dentro del contenedor, se coloca en el contenedor padre grid.
nota:
Valores de justify: [start | end | center | space-between | space-around | space-evenly]
Valores de align: [start | end | center | space-between | space-around | space-evenly | baseline | stretch]

Grid placement

El grid placement es el algoritmo que utiliza css grid para saber dónde colocar los elementos en el grid.
Existen dos maneras de clasificar al grid en el código, el grid implícito y el grid explícito, el explícito es el que colocamos y definimos intencionalmente en el código, el implícito es el que se define automáticamente ya que no fue declarado en el código, pero es necesario para que funcione correctamente, a esto se le atribuye el anexo de filas o columnas no declaradas las cuales no tienen propiedades.

Auto

grid-auto-rows: #px; define las medidas para filas que se creen de forma implícita o automática.
En el ejemplo a la nutria se le colocó un grid-row: 3 / 6; (ocupa de la grid line horizontal 3 a la 6), pero solo se definió un grid-template: repeat(2, min-content) / repeat(3, min-content); (2 filas estrechas con 3 columnas estrechas). Para colocar a la nutria en esas filas, el navegador definió 3 filas más, normalmente medirían 0 de largo por lo que no se notarían, pero el grid-auto-rows: 1.3em; les proporciona una medida definida a cada fila y la nutria puede visualizarse con las filas especificadas.
grid-auto-columns: #px; define las medidas para columnas que se creen de forma implícita o automática.
En el ejemplo a la nutria se le colocó un grid-column: 3 / 6; (ocupa de la grid line vertical 3 a la 6), pero solo se definio un grid-template: repeat(3, min-content) / repeat(2, min-content); (3 filas estrechas con 2 columnas estrechas). Para colocar a la nutria en esas columnas, el navegador definió 3 columnas más, normalmente medirían 0 de ancho por lo que no se notarían, pero el grid-auto-columns: 1.1em; les proporciona una medida definida a cada columna y la nutria puede visualizarse con las columnas especificadas.

Flujo

grid-auto-flow: [row | column] [dense]; con el primer valor se indica el flujo de los tracks, con el segundo valor se indica si los espacios vacíos entre los tracks se pueden ocupar por tracks aunque se altere su secuencia u orden del HTML.
