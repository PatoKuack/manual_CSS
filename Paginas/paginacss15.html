Variables CSS.

Las variables son un valor que se almacena en memoria para después ser utilizado. Las variables CSS (o también conocidas como propiedades personalizadas) son case sensitivity (distinguen entre mayúsculas y minúsculas) y son muy utilizadas en los colores principales en un sitio web.

Uso de variables.

Declaración.

Las variables se declaran en un selector llamado :root{}

Dentro del selector se colocan las variables empezando por su nombre, el nombre de la variable debe iniciar con dos guiones medios (-), puede ser cualquier tipo de notación y después se colocará el valor de la variable, ejemplo:

:root {
 --text-color: peru;
 --borderColor: lime;
}

Llamado.

La forma de mandar a llamar una variable es con la palabra var seguida de un bloque de paréntesis en donde se coloca el nombre de la variable, ejemplo:

.body {
 color: var(--text-color);
}

Valores que reciben.

  • Colores en HSL, RGBA, HEX o cualquier tipo de color (como el predefinido "pink").
  • Pasar varios valores como si se estuviera asignando el valor (1px solid red).
  • Cadenas de texto ('hello lady').
  • Variable dentro de otra variable (var(--txtColor)).
  • Múltiples valores almacenados en una sola (50%, 75%, 100%).

Fallback.

Es un valor que se utiliza al momento que algo falla, es decir un valor de respaldo si la variable falla.

:root{
 --txt-clr: blue;
 --bg-clr: green;
}
.body {
 color: var(--txt-clr, var(--bg-clr));
 color: var(--txt-clr, red);
}

Operaciones.

Para crear operaciones en CSS se utiliza la función calc() la cual guarda un valor, sino que hace un cálculo en tiempo real y lo utiliza.

:root{
 --min-size: 3rem;
 --size: calc(var(--min-size) * 2);
}

Conceptos.

Scope.

Es al ámbito o contexto en el que existe una variable y que en él mismo puede ser utilizada, es decir, lo que se crea y existe únicamente dentro de un determinado bloque de código.

Existen 3 tipos de scope en las variables CSS:

Global.
Se declaran en el :root{} o al inicio del archivo y es accesible en cualquier parte del documento o página web sin importar que tan profunda es la estructura.
DOM.
Si comparten estructura o relación en el documento HTML entonces compartirán las mismas variables en el CSS.
Local.
Declaración en la línea en donde se declara la etiqueta HTML en el documento HTML. Estas variables solamente existirán en estas etiquetas y en los hijos que desciendan de ella.

Hoisting.

Es la acción que hace el lenguaje para reordenar la declaración de variables con respecto al uso, es decir, si se utiliza una variable y después se crea esa variable usada, el lenguaje colocará la declaración (creación) de la variable antes de su uso al momento de compilar el código.

Herencia.

Es la habilidad que tienen los elementos de obtener los valores o estilos de sus ancestros.

Cascada.

Es la manera que tiene el lenguaje de sobre escribir valores o estilos a aquellas propiedades que ya tenían un valor, pero se vuelven a declarar con valores diferentes.

RWD.

Responsive Web Designe es la adaptación de la página web o aplicación al tamaño del dispositivo o de la ventana a través de condiciones llamadas media queries. Gracias a las variables CSS, el RWD se puede realizar de una forma más fácil, sin utilizar JavaScript para calcular tamaños o escalas.

SASS.

Es un preprocesador de CSS que agrega sintaxis relacionada con la programación para facilitar la creación de estilos y trabajar de una manera más rápida y escalable (darle fácil mantenimiento al programa).

Las diferencias entre las variables CSS y las variables SASS son las siguientes:

  • La forma de declarar una variable es con el signo monetario: $name: value;
  • Scope de bloque, en SASS se pueden declarar selectores con sus respectivos bloques de estilos dentro de otros bloques de estilo de selectores ajenos y pueden tomar las variables del selector externo.
  • Se pueden interpolar variables, es decir, utilizar las variables como selectores o transformarlas en otro tipo de valor.
  • En las operaciones con SASS se obtiene el valor a diferencia de CSS que únicamente se calcula.
  • En CSS se pueden reasignar valores dentro de selectores y siguen existiendo esos valores aún sin ser utilizados en cambio en SASS no se compilan esos valores reasignados si no se utilizan.

Creación de temas.

Cuando se crean temas en las páginas como las típicas versiones normal en donde el fondo es blanco u oscuro en donde el fondo es negro, las variables son muy importantes ya que cambian sus valores al modificar estos temas.

En la creación de temas se deben tener claros que valores se van a cambiar en los componentes cuando el usuario pase de un tema al otro. También se debe tener una paleta de colores definida para facilita el desarrollo del código.

Se puede hacer la asignación de la clase con JavaScript o con un media query especial que detecta el tema del navegador.

@media ( prefers-color-scheme : [ no-preference | light | dark ] ) { ... }

Este media query especial activa su contenido cuando se cambia el tema del navegador (todos los navegadores le dan soporte a esta media query)

:root {
 --bd: white;
}

@media (prefers-color-scheme : dark) {
 :root {
  --bg: black;
 }
}

.body {
 background-color: var(--bg);
}
