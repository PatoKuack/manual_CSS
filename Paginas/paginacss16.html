Stacking context.

El contexto de apilamiento es la interpretación de los elementos HTML en el eje imaginario z (apilamiento de los elementos), es decir, es la manera en que el navegador interpreta como se pone un elemento sobre otro. Existen 4 niveles de apilamiento que ayudan a determinar que elementos van encima de otros:

  1. Elementos con nivel de apilamiento negativo (se colocan hasta abajo de los elementos).
  2. Elementos con position static (todos los elementos con el valor de posicionamiento por defecto).
  3. Elementos con nivel de apilamiento 0 o auto, este nivel de apilamiento se le asigna automaticamente a elementos cuando se le asigna un posicionamiento diferente a static.
  4. Elementos con nivel de apilamiento mayor a cero (se colocan hasta arriba de los elementos).

Cuando todos los elementos tienen el mismo nivel de apilamiento, la superposicón de un elemento con respecto a otro se decide con el flujo en que se declaran los elementos en el archivo html, el primer elemento puede ser posicionado por debajo del segundo que puede ser el hijo o el hermano siguiente.

z-index.

Permite cambiar el nivel de apilamiento de un elemento, siguiendo la siguiente escla:

  • z-index: -#; Números negativos se quedan hasta abajo, entre menor sea el numero negativo más abajo queda el elemento.
  • z-index: 0; Numero cero es como un valor por defecto.
  • z-index: #; Números positivos se colocan hasta arriba, entre mayor sea el valor más arriba queda el elemento.

Es reccomendable dar valores en el z-index de decenas (como 10, 20, 30,...) por si se requiere posteriormente colocar un elemento entre dos elementos con z-index ya que no se pueden usar decimales como valor de esta propiedad (como 2.5 o 4.1).

Esta propiedad no es única de la propiedad position, hay elementos que pueden utilizarla como hijos de flex y grid containers, elementos con opacity, filtros, mix-blend-mode o perspectiva con valor diferente al inicial.

Cuando se le da un nivel de apilamiento (con z-index) a un elemento padre, sus hijos no podrán subponerse a él aún cuando tengan un nivel de apilamiento menor que el padre.

pointer-events.

El problema cuando se manejan distintos niveles de apilamiento y los elementos se ponen unos sobre otros, es que el mouse solo detecta al elemento que esta hasta arriba (el de mayor nivel de apilamiento), por lo cual no logra acceder a los de abajo.

Este problema se soluciona con la propiedad pointer-events: none; el cual, con su valor none indica a un elemento si es detectable o no ante el mouse, dando paso a que el mouse pueda seleccionar o interactuar con elementos inferiores.
