Listas y contadores.

Listas

Las listas por defecto tienen estilos, las ordenadas tienen una numeración y las no ordenadas tienen viñetas, existe la propiedad list-style-type: ___; con la cual se pueden modificar estos estilos (tiene un shorthand: list-style: _;), a continuación se muestran los valores de esta propiedad:

none
Elimina los estilos de las listas.
disc
Coloca una viñeta en forma de disco.
circle
Coloca una viñeta en forma de círculo.
square
Coloca una viñeta en forma de cuadrado.
decimal-leading-zero
Coloca números decimales empezando eventualmente por ceros (por ejemplo: 01, 02, ..., 98, 99).
upper-roman
Coloca números romanos en mayúscula (I, II, III, IV etc.).
Existe su versión con "lower-roman" o con varios alfabetos como griego o latín.

Las listas también pueden adquirir imágenes en vez de viñetas o números con la propiedad list-style-image: url;, solo que las imágenes no se pueden redimencionar, por lo que deben tener el tamaño adecuado, aunque la mejor manera de colocar imágenes en los estilos de las listas es colocando pseudo-elementos vacíos como se vió anteriormente.

Otra propiedad muy útil es list-style-position: [ inside | outside ]; el cual indica si las viñetas o números de los items se colocan adentro de su entorno o afuera de su entorno, por default estan afuera del entorno de los items por lo que si se le da un estilo de fondo al item, la viñeta o numero no recivirán ese estilo.

Existen dos formas de modificar la viñeta de las listas, la unica desventaja con estas dos opciones es que no todos los navegadores les dan soporte todavía (actualmente del año 2021):

list-style-type: symbols(...);
Esta propiedad tiene varias funciones para ordenar o modificar las viñetas (pueden colocarse emojis). Solo es soportado por 1 navegador actualmente.
li::marker{content: "-";}
Este pseudo-elemento cambia las viñetas de las list items colocando en su lugar lo que se coloca en la propiedad content (pueden colocarse emojis). Es soportado por varios navegadores actualmente.

Contadores.

Esos contadores son para las ordenar listas ordenadas o no ordenadas y tienen varias ventajas:

counter-reset: name's-counter #;

Esta propiedad se coloca en el bloque de padre de los elementos a los que se les aplicará el contador y sirve para reiniciar y crear contadores.

Se declara colocando como primer valor el nombre que se le dará al contador y como segundo valor (opcional) se especifica después de qué número debe empiezar el conteo de los elementos hijo.

Es recomendable que se eliminen las viñetas o numeros de las listas para que tengan buena presentación (list-style-type: none;).

counter-increment: name's-counter #;

Esta propiedad se coloca en el bloque de los elementos hijo.

Indica el conteo de incremento de la numeración de los elementos hijo, es decir, indica si los elementos se contarán de 2 en 2 o de 10 en 10.

Se declara colocando como primer valor el nombre que se le ha dado al contador y como segundo valor (opcional) la unidad de incremento.

content: counter(name's-counter, upper-roman) '.- ';

La función counter se coloca en el bloque de los elementos hijo con el pseudo-elemento ::before y como valor de la propiedad content.

Indica el tipo de viñeta o numeración de los items y gracias al pseudo-elemento puede añadir contenido para separar el item del pseudo-elemento o para agregarle más contenido a las numeraciones.

Se declara colocando como primer parámetro el nombre que se le ha dado al contador y como segundo parámetro el tipo de numeración (alguno de los valores de list-style-type).

Contadores anidados.

Los contadores anidados ayudan a darles un estilo diferente a los sub-items de las listas anidadas.

ul {
 list-style: none;
 counter-reset: mi-lista;
}
li {
 counter-increment: mi-lista;
}

li::before {
 content: counters(mi-lista, '.', decimal) ' ';
}

Se pueden colocar 2 counters en una sentencia, ejemplo:

p::before {
 content: counter(chapter) '.' counter(sub-chapter) ' ';
}
// "chapter" representa el contador el padre y "sub-chapter" representa el contador del elementoactual.
