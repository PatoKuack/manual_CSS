Formularios.

En los elementos de los formularios se tiene por defecto asignada la propiedad -moz-appearance: ; que tiene muchos valores para cada tipo de dato, pero su principal uso en CSS es modificar el checkbox.

input[type="checkbox"] {
 -moz-appearance: none;
}

input[type="checkbox"]::before {
 content: ' ';
 display: block;
 width: .5rem;
 height: .5rem;
 border: 1px solid blue;
}

// checked es una pseudo-clase para avisar si el elemento esta activado o no. input[type="checkbox"]:checked + label {
 background-color: tomato;
}

Las etiquetas textarea que hay en los formularios tienen por defecto una propiedad llamada resize: [ both | none | vertical | horizontal ]; que sirve para modificar las dimensiones del elemento contenedor directamente desde la página web. Esta propiedad se puede utilizar para cualquier elemento, aunque su principal uso es en los textarea.

Los valores de resize tienen los siguientes efectos:

  • none no permite redimensionar los elementos
  • both permite redimensionar el elemento vertical y horizontalmente.
  • vertical permite redimensionar el elemento únicamente de forma vertical.
  • horizontal permite redimensionar el elemento únicamente de forma horizontal.

Pseudoclases para formularios.

:focus
Se utiliza generalmente en elementos seleccionables como en los formularios. Aplica un estilo al elemento al que se le tiene el foco.
:focus-within

Se utiliza para aplicar un estilo al elemento padre del elemento seleccionado. Es la única pseudo-clase que permite seleccionar a un padre de forma directa.

Incluso se podría aplicar un estilo a los hermanos del contenedor padre con selectores.

:placeholder-shown
Se utiliza en el input y le aplica un estilo mientras se esté mostrando el placeholder.

En los formularios existen campos con los atributos require para que el formulario no sea mandado sin completar los campos con este atributo. Para este atributo existen dos pseudo-clases:

:required
Les aplica un estilo a los elementos con el atributo require.
:optional
Les aplica un estilo a los elementos sin el atributo require.

También existen los atributos readonly que evita que un campo del formulario sea modificado (los botones tienen este atributo por default). Para este atributo existe dos pseudo-clases:

:read-only
Aplica un estilo a los elementos con el atributo readonly (en los que nos e puede escribir).
:read-write
Aplica un estilo a los elementos sin el atributo readonly en los que se puede escribir.

Otro atributo es disable que desactiva un campo, algunos navegadores aplican "readonly" a los campos con "disable". Para este atributo existen 2 pseudo-clases:

:disabled
Aplica un estilo a los elementos con el atributo disable.
:enabled
Aplica un estilo a los elementos sin el atributo disable.

Cuando un campo del formulario no es válido, es porque no cumple con los caracteres mínimos, los máximos, con algún @ en los campos de e-mail, etc. Para los caracteres inválidos existen 2 pseudo-clases:

:valid
Aplica un estilo a los elementos válidos.
:invalid
Aplica un estilo a los elementos inválidos.

Existe un pseudo-elemento para los placeholder que funciona como la pseudo-clase:

::placeholder
Aplica un estilo a los campos con place holder desactivando el estilo cuando el usuario escribe sobre el campo ya que al hacer eso el placeholder desaparece.
