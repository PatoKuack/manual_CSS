Filtros.

Se le pueden aplicar filtros a las imágenes, por medio de la propiedad filter, colocándole como valores, funciones especiales de CSS.

Filtros de colores.

filter: grayscale();
Aplica un degradado de grises sacando el promedio de tonalidad a cada pixel (ejemplo: rgb(200, 100, 60) = rgb(120, 120, 120)), se le puede colocar como parámetro un valor de 0 a 1, hace caso omiso a valores menores a 0 y mayores a 1. También se le puede colocar valores del 0% al 100%, si se omite el valor en el parámetro, automáticamente se aplicará un 100%.
filter: sepia();
Aplica una conversión de colores a escalas de tipo marrón, se le puede colocar como parámetro un valor de 0 a 1, hace caso omiso a valores menores a 0 y mayores a 1. También se le puede colocar valores del 0% al 100%, si se omite el valor en el parámetro, automáticamente se aplicará un 100%.
En javascript se puede editar este filtro para modificar la conversión a la escala marrón.
filter: invert();
Aplica una inversión de colores colocándole a cada pixel su contraparte de color restando a sus tonalidades de rgb 255 (ejemplo: rgb(255-50, 255-80, 255-200)), se le puede colocar como parámetro un valor de 0 a 1, hace caso omiso a valores menores a 0 y mayores a 1. También se le puede colocar valores del 0% al 100%, si se omite el valor en el parámetro, automáticamente se aplicará un 100%.
Cuando se le coloca un .5 o 50% en el parámetro, la imagen se ve completamente gris ya que todos los colores se colocan en una tonalidad neutra que resulta ser gris.
filter: hue-rotate();
Funciona con la rueda de colores posicionando cada pixel en el grado de su tonalidad rotando su tono de acuerdo a los grados del parámetro, se le puede colocar como parámetro valores en radianes, numero de vueltas (1turn) o grados de 0deg a 360deg, si se coloca un número mayor vuelve a contar desde 0, por ejemplo, un valor de 361deg lo tomaría como 1deg. También acepta números negativos los cuales resta de 360, por ejemplo, un valor de -100deg lo tomaría como 260deg.
// Para una animación de colores:

.imagen {
 filter: hue-rotate(0);
 animation: hue 3s linear infinite alternate;
}

@keyframes hue {
 to {
   filter: hue-rotate(1turn);
 }
}
filter: brightness();
Maneja la luminosidad de los colores, el valor por default es 100% o 1, se le puede colocar como parámetro un valor mayor o igual a 0, hace caso omiso a valores menores a 0. También se le puede colocar valores mayores o igual a 0%, si se omite el valor en el parámetro, automáticamente se aplicará un 100%.
filter: contrast();
Hace más marcada las delimitantes entre las áreas de las luces y sombras, el valor por default es 100% o 1, se le puede colocar como parámetro un valor mayor o menor a 1, o mayor o menor a 100%. El valor de 0% o 0 elimina todo el contraste entre las áreas oscuras o claras haciendo que la imagen sea gris completamente.
filter: saturate();
Aumenta la pureza de cada color, el valor por default es 100% o 1, se le puede colocar como parámetro un valor mayor o menor a 1, o mayor o menor a 100%. El valor de 0% o 0 elimina toda la saturación haciendo que la imagen se convierta en una escala de grises.

Filtros de transformación.

filter: blur();
Mezcla los pixeles entre sí de acuerdo con el valor que se le proporcione en el parámetro, recibe como parámetro una unidad de medida (como px) y no recibe valores negativos, en sí es el perímetro el que utiliza para mezclar los píxeles del elemento.
filter: opacity();
Existe ya una propiedad opacity la cual sirve para elementos estáticos como fotos, pero el filtro opacity sirve para elementos animados (utiliza aceleración de hardware), recibe valores mayores a 0 como parámetro.
filter: drop-shadow();
Se aplica a elementos .svg o .png dándoles una sombra al contorno de sus contenidos usando el canal alfa(transparencia) para dar sombra en cambio la propiedad box-shadow le crea una sombra a la figura perimetral del elemento, recibe como parámetros 4 valores, el traslado de la sombra en X, en Y, el nivel de difuminado de la sombra y el color de la sombra.

Se pueden colocar múltiples filtros en la misma propiedad, cada filtro trabaja con lo que se le va mostrando y no con el elemento base, así que el orden en que las funciones se colocan dentro de la propiedad filter influye en el resultado de la imagen.
