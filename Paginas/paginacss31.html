Responsive Web Design.

Es un conjunto de metodologías de buenas prácticas de patrones de diseño de interfaces, lo que busca es que cualquier página web se vea bien en cualquier tamaño de dispositivo.

El responsive web design se basa en tres principios: los grids fluidos, las imágenes flexibles y los media queries.

Un sistema de grid es colocar contenido dentro de filas y columnas permitiendo manejar fácilmente el layout (geometría de la página). Estos sistemas de grid se vuelven fluidos cuando su volumen se define por medio de porcentajes ya que se podrá adaptar fácilmente el volumen del tamaño del sistema grid dependiendo del contenedor del sistema grid.

Las imágenes se vuelven flexibles cuando se les coloca un máximo de ancho evitando que aumenten su tamaño más de lo que deben y manteniendo un tamaño mínimo dependiendo de su contenedor.

Las medias queries (consulta de medio) son consultas de visualización de la web denominados medios, es donde se muestra el contenido de la página o aplicación.

Viewport.

Es toda la parte visible del navegador para mostrar la página web, cuando la página es más grande que el viewport aparece la barra de desplazamiento la cual no pertenece al viewport.

En un HTML se declara el viewport usando la etiqueta <meta name="viewport" content="width=device-width, initial-scale=1.0">

El atributo name indica que se aplicará un viewport a esa página y con el atributo content se le colocan las siguientes opciones.

width=device-width
Indica que el navegador debe tomar como ancho de la página el ancho del dispositivo.
user-scalable=no|yes
Indica si se puede o no usar el zoom en la página, se recomienda siempre colocarlo en "yes".
initial-scale=1
Indica cual será la escala inicial, es recomendable colocarle una escala de 1 que es la escala default, sino el contenido se presentara en escala reducida (.5) o maximizada (2).
maximum-scale=2
Indica el zoom máximo que el usuario puede usar, si se coloca 2 será de 200, 3 es de 300, etc.

El viewport es diferente a la resolución de la pantalla de un dispositivo o en otras palabras, 1 pixel del viewport es diferente de un pixel real. Con la instrucción devicePixelRatio escrita en la consola del navegador se puede obtener el número de pixeles reales que hay por cada pixel de viewport.

devicePixelRatio
// Si obtenemos por ejemplo 1.25, entonces por cada pixel de viewport hay 1.25 pixeles reales de la pantalla.

Media Queries.

Las Medias Queries o Consulta de Medios son un módulo CSS3 que permite adaptar la representación del contenido a características de un dispositivo (como una impresora, una pantalla de smartphone o de Tv); y consiste en definir al menos una consulta (@media), donde se define un tipo de medio (media type) y una o más características o parámetros específicos (media features) como la resolución de la pantalla o el ancho del viewport del navegador. Varios parámetros se pueden combinar utilizando los operadores AND y OR.

Estas consultas funcionan a manera de condicionales, cuando el tipo de medio (si se especifica) coincide con el dispositivo en el que se está mostrando el documento y todas las expresiones en el media query son "verdaderas" se aplicaran los estilos CSS que se redefinan para esa condición siguiendo las reglas usuales de cascada; por tanto, se recomienda un enfoque 'mobile first', generar los 'Media queries' apropiados y en el orden apropiado

Media Type

all
Apto para todos los dispositivos.
print
Destinado a material impreso y visualización de documentos en una pantalla en el modo de vista previa de impresión.
screen
Es el más comúnmente utilizado, está destinado principalmente a las pantallas.
speech
Destinado a sintetizadores de voz.
 @media screen{
  body {
   color: black;
  }
  // Se indica que se utilizará el RWD en una pantalla (screen).
 }

Media Features

min-width | max-width
Define el mínimo de ancho o el máximo de ancho del elemento.
orientation: landscape | portrait
landscape: cuando el ancho de la pantalla es mayor al alto de la misma. (ancho>alto)
portrait: cuando el ancho de la pantalla es menor al alto de la misma. (ancho<alto)
aspect-ratio
Según la proporción o razón del ancho y alto de la pantalla (ancho:alto), puede pedir una proporción especifica o agregarle un "min" o "max" para un mínimo o máximo, ejemplo: @media screen and (min-aspect-ratio: 4/3){...}
 @media screen and (min-width:1024px) and (orientation: landscape) {
  body {
   color: black;
  }
  // Se le anexa una condición para que el contenido de la media query se aplique a los dispositivos que sean pantallas, que midan mínimo 1024px de tamaño horizontal y que su orientación sea horizontal (que la parte horizontal sea mayor que la vertical).
 }

Operadores lógicos.

Se deben utilizar los operadores lógicos para unir las características de las medias queries.

and
Este operador es usado para colocar juntas múltiples funciones multimedia.
Lista separada por comas
Las listas separadas por comas se comportan como el operador or. Cuando utilice una lista separada por comas y alguno de los queries retorna verdadero, el estilo o la hoja de estilos será aplicada. Cada query en una lista separada por comas es tratado como una query individual y cualquier operador aplicado a un medio no afectara a los demás.
 body{ background: cyan; }
 @media (min-width: 700px), (max-width: 1024px);{
  body{ background: lime; }
 }
not
Este operador aplica a todo el query y retorna verdadero si es posible y sino retorna falso. Un not negara un query si es posible pero no a todos los query posibles si están ubicados en una lista separada por comas. No puede ser usado para negar un query individual, solo para un query completo.
 @media (not (print and (min-width:800px))), screen and (max-width:1024px) {
  // Esto es igual a:
@media not print and (min-width:800px), screen and...{}
  body{ background: lime; }
 }
  // Si la página se visualiza en un formato de impresión y mide mínimo 800px retornará verdadero y se aplicará el query, pero también se puede aplicar el query si se cumple la segunda opción que pide que sea una pantalla con un máximo de 1024px.
only
Este operador previene que navegadores antiguos que no soportan queries con funciones apliquen los estilos asignados, ejemplo:
<link rel="stylesheet" media="only screen and (color)" href="Ejemplo.css" />

Metodologías.

Mobile first
El mobile first (o versión móvil primero) es la metodología más recomendada y es el enfoque en el diseño y desarrollo de sitios web orientado a comenzar con la información y características más importantes para las versiones móviles, para luego añadir más a medida que en la pantalla se hace más grande en tablets, laptops y escritorio.
Desktop first
El desktop first (escritorio primero), es como se realizaban los primeros sitios web responsive, en donde primero se desarrolla la versión de escritorio, y luego, esta se adapta para mostrar el sitio web correctamente en pantallas más pequeñas.
Content first
Consiste en considerar desde el principio el proceso de creación de contenido del proyecto; para organizar y diseñar mejor los componentes dentro de un layout, así como los elementos del diseño gráfico propiamente; para cuando se entrega al cliente y se implementa en producción no se rompa o existan comportamientos inesperados. Adoptando el enfoque Content-First se coloca el contenido en el centro del proceso de diseño web y la experiencia de usuario. No es una metodología excluyente de las otras dos metodologías; se trata de que el contenido y diseño trabajando juntos.

Unidades de medida.

Unidades absolutas.

Define el valor de la propiedad de un elemento en referencia a una medida real, inalterable y no dependiente de su contenedor.

Unidades relativas.

Definen su valor real en relación con otra medida determinada anteriormente (en la propiedad de su elemento contenedor o ancestro); por tanto, van a cambiar dependiendo las circunstancias.

Las unidas relativas más utilizadas son el porcentaje (%), 'em', 'rem', 'vw' y 'hw'. Pero es de recordar que, al momento de renderizar, el navegador siempre expresará las unidades relativas en píxeles (la unidad absoluta).

* Para lograr un RWD es recomendable el uso de medias queries, pero no son necesariamente obligatorias, basta un uso adecuado e inteligente de las unidades de medidas; evitando las unidades absolutas e implementando más las unidades relativas como min/max-width o min/max-height;.

Unidades relativas de medida.

  • Porcentaje (%)

    El porcentaje confiere un tamaño a un elemento con referencia al de su elemento padre.

    Un valor del 100% ocupará todo el espacio disponible del que lo contiene y 50% ocupará la mitad del tamaño disponible de su elemento contenedor.

    Puede ser usada en cualquier propiedad CSS que requiera valores de longitud.

  • em

    Toma el valor del tamaño de fuente del contexto definido más próximo, sea del elemento padre, su hermano directo superior o hasta del mismo elemento, por tanto, en este último caso, el orden de la declaración dentro la regla CSS es importante.

    1em = 100% del tamaño de fuente en el contexto, por tanto, va a variar dentro del mismo documento

    El uso de 'em' no es recomendable como unidad de medida para layout; pero sí para componentes internos (como márgenes, padding, etc).

  • rem

    Toma el valor del tamaño de la fuente por default en el ancestro más lejano, en un documento HTML la etiqueta es <html> o <body> y en la mayoría de los navegadores de forma predeterminada es 16px.

    1rem = 100% del tamaño de fuente del root <html>

    El uso de 'rem' es recomendable como unidad de medida para layout (como margin o width o height de un elemento).

  • Viewport

    Son unidades de medida relativas en función al tamaño del Viewport.

    1 vw = Viewport weight = 1% del ancho del Viewport.
    1 vh = Viewport height = 1% del alto del Viewport.
    1 vmin = Viewport mínimo = 1% del lado menor del viewport.
    1 vmax = Viewport máximo = 1% del lado mayor del viewport.

    Las unidades del Viewport equivalen a la centésima parte con respecto al Viewport; y no depende de su contenedor, como en la unidad relativa porcentual (%).

    El viewport es el área visible de la página web del usuario. Debido a esto, el scroll vertical genera algunos inconvenientes.

    // *** Cuando un elemento tiene un ancho de 100vw. ***
     footer {
      padding: 10vmin 0;
     // El padding top y bottom del footer será de 10% del lado menor del viewport, se adaptará dependiendo del viewport.
      width: 100vw;
     }
     // *** Y se genera un scroll vertical por el gran contenido de la página, el contenido horizontal se desbordará porque se suma el 100% del viewport + el ancho del scroll vertical. ***
     :root {
      --scrollBar: 13px;
     }
     footer {
      padding: 10vmin 0;
      width: calc(100vw - var(--scrollBar));
     }
     // *** EN JAVASCRIPT ***
     const getScrollBarWidth = () => innerWidth - document.documentElement.clientWidth;
     // Se resta el ancho interno menos el ancho de la página HTML.
     document.documentElement.style.setProperty('--scrollBar, getScrollBarWidth()');
     // Con el comando setProperty se sustituye la propiedad indicada en el primer parámetro por el valor indicado en el segundo parámetro y así se actualizará el ancho del footer automaticamente.

    Esta actualización automática es importante porque en dispositivos móviles el scroll no ocupa espacio del viewport a diferencia de las laptops o computadoras de escritorio.
