Breackpoints

Los breakpoints o puntos de quiebre, son puntos en los que cambian el diseño basado en el viewport del dispositivo (major breakpoint) o por ajustes del contenido (minor breakpoint), implementando media queries y mostrando el mejor diseño posible al usuario.

  • Major breackpoints: Representan cambios mayores en el diseño, componentes o layout.
  • Minor breacpoints: Representan cambios menores muy específicos, donde el contenido necesita algún ajuste de diseño.

Es una muy buena practica utilizar SASS para crear un mapa de breackpoints ya que se utilizan medidas especificas para implementar el responsive web designe en cada tamaño de dispositivo, el siguiente ejemplo muestra algunos tamaños masyormente utilizados pero no existe un estandar:

Para pantallas pequeñas se definen medidas de 320px o 360px o 480px
Para pantallas medianas se definen medidas de 640px o 768px
Para pantallas grandes se definen medidas de 960px o 1024px
Para pantallas extra grandes se definen medidas de 1200px o 1280px
Para pantallas super extra grandes se definen medidas de 1440px o 1600px

// *** EJEMPLO CON SASS ***
 $breakpoints: (
  small: 320px,
  medium: 640px,
  large: 1024px,
  xlarge: 1200px,
  xxlarge: 1440px,
 );
 @mixin rwd($bp){
  @media screen and (min-width: map-get($breakpoints, $bp))  {
   @content;
  }
 }
 body{
  background: red;
  @include rwd(small){
   background: orange;
  }
  @include rwd(medium){
   background: yellow;
  }
 }
// *** EJEMPLO CON CSS ***  body {   background: red;  }  @media screen and (min-width: 320px) {   body {    background: orange;   }  }  @media screen and (min-width: 640px) {   body {    background: yellow;   }  }

Video responsive.

Para visualizar correctamente un video en diferentes tamaños de viewport, el contenedor (frame) debe guardar una proporción estándar de 16:9 entre su ancho y alto (relación de aspecto o aspect ratio) a lo largo de las diferentes medias queries. Si 16 es 100%, entonces 9 es 56.25% por lo que se utiliza 56.25% como referencia de proporción.

El formato 16:9 codifica en las resoluciones comúnmente más utilizadas: 240 píxeles: 426 x 240 360 píxeles: 640 x 360 480 píxeles: 854 x 480 720 píxeles: 1280 x 720 1080 píxeles: 1920 x 1080.

 .video {
  height:    0;
  padding-bottom: 56.25%;
  position:    relative;
// Como el padding siempre es relativo al ancho, empuja la altura del elemento que por defecto es 0, ésto hace que se ajuste automaticamente con el viewport o contenedor.
  > iframe, > video {
   position: absolute;
   width: 100%;
   height: 100%;
// Los hijos directos que sean iframe o video, obtienen los atributos.
  }
 }

Contenedor responsive formato 16:9 utilizando display grid:

 .video {
// Se crea el grid separando una proporción de 16:9.
  display: grid;
  grid-template-columns: repeat(16, 1fr);
  grid-template-rows: repeat(9, 1fr);
  background: blue;
  position: relative;

// Se vuelven cuadradas las filas y columnas.
  &::before {
   content:"";
   display: block;
   padding-bottom: 100%;
  }

// Inserción de video.
  > iframe, video {
   position: absolute;
   width: 100%;
   height: 100%;
  }
 }

Imágenes responsive.

Si bien las imágenes cargadas en un diseño RWD deben adaptarse al tamaño del contenedor responsive, estas también deben optimizar su carga en la web según el tamaño de viewport del dispositivo; partiendo del principio de que no es necesario mostrar una imagen de gran tamaño en una pantalla significativamente más pequeña que el tamaño para el que fue diseñada, hacerlo genera costes de ancho de banda, tanto para el usuario final como para la carga de la web.

Las imágenes adaptables son una forma de ofrecer al navegador varias versiones de imágenes en diferentes archivos (ya sea que muestren lo mismo), pero que contengan diferentes resoluciones, para que la carga de estas imágenes sea según las características y tamaño de pantalla del dispositivo y/o adecuadas para diferentes asignaciones de espacio y orientación.

Una de las formas para usar imágenes con RWD es colocar la imagen como background dentro de un contenedor adaptable.

 .image{
  background-image: url(tu-imagen-deMenorResolucion.jpg);
  background-posittion: center;
  background-size: contain;
 }

 @media screen and (min-width: 760px) {
  .imagen {
   background: url(tu-imagen-deMayorResolucion.jpg) center/cover;
  }
 }

Otra forma de utilizar RWD en imágenes es utilizando el elemento HTML <picture>, el cual permite sin necesidad de CSS, indicar al navegador qué imagen mostrar en cada tamaño de pantalla. Esto requiere de dos elementos anidados, <source> e <img>.

Los elementos <source> trabajan con base en una media query (atributo media) y un recurso que se adapta a la condición (atributo srcset).

El elemento <img> contiene la ruta a un recurso que solo cargará si los <source> fallan.

El orden de la declaración de los elementos <source> en base al atributo media, es muy importante, pues solo toma el primero que cumpla una condición obviando al resto y no existe un efecto cascada. El orden de las condiciones es importante.

 <picture>
  <source media="(min-width:1000px)" srcset="./img/extraLarge.jpg">
  <source media="(min-width:800px)" srcset="./img/large.jpg">
  <source media="(min-width:600px)" srcset="./img/medium.jpg">
  <img src="./img/small" alt="Texto Alt.">
 </picture>

El elemento <picture> también permite cargar imágenes según si el navegador soporta o no ciertos formatos; como el caso de webp (un formato de imagen moderno que proporciona una compresión optimizada), pero puede no ser soportada por algunos navegadores o versiones antiguas de los mismos.

 <picture>
  <source media="(min-width:800px)" srcset="./img/large.webp" type="image/webp">
  <source media="(min-width:800px)" srcset="./img/large.jpg" type="image/jpeg">
 </picture>
// De esta forma si el navegador no soporta el tipo de formato webp salta al siguiente elemento <sourse> que al ser un formato estándar como jpg lo renderiza.

Otra forma es declarando el atributo srcset el cual puede ser declarado de forma adicional en el elemento HTML <img> para proporcionar varias imágenes adicionales, junto con unos indicadores y ayudar al navegador a elegir la correcta, según criterios internos del navegador basándose en la información que le suministramos, sin necesidad de CSS.

Cada conjunto de información, está formado por el nombre&extensión del archivo, el ancho intrínseco de la imagen en píxeles y separado de la anterior por una coma.

La unidad para el ancho intrínseco de la imagen, se representa con w, y no con px. Siendo este el tamaño real de la imagen.

Navegadores como Chrome renderizaran la imagen de mayor resolución que hayan logrado descargar en caché, según nuestras indicaciones.

 <img
  srcset="img/medium.jpg 500w, img/extraLarge.jpg 1000w"
  src="defaul.jpeg"
  alt="Texto Alt."
 >
// En src se coloca la imagen que aparecerá por default si es que el navegador no encuentra las de srcset.

Cuando el navegador descarga la imagen mas grande, no descarga las imágenes más pequeñas a esa porque si la imagen grande ya esta en el cache del navegador, no descarga una imagen menor porque no es necesario utilizar una con menor resolución si ya se tiene una con mejor calidad.

Temas responsive.

Cuando se crean temas en las páginas como las tipicas versiones normal en donde el fondo es blaco u oscuro en donde el fondo es negro, las variables son muy importantes ya que cambian sus valores al modificar estos temas.

En la creación de temas se deben tener claros que valores se van a cambiar en los componentes cuando el usuario pase de un tema al otro. También se debe tener una paleta de colores definida para facilita el desarrollo del código.

Se puede hacer la asignación de la clase con JavaScript o con un media query especial que detecta el tema del navegador.

@media ( prefers-color-scheme : [ no-preference | light | dark ] ) { ... }

Este media query especial activa su contenido cuando se cambia el tema del navegador (todos los navegadores le dan soporte a esta media query)

:root {
 --bd: white;
}

@media (prefers-color-scheme : dark) {
 :root {
  --bg: black;
 }
}

.body {
 background-color: var(--bg);
}
