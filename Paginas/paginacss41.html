SASS

Sass es un preprocesador escrito en Ruby en el año 2006.

Un preprocesador es un lenguaje que permite generar CSS a través de una sintaxis similar, pero aprovechando características de la programación.

Algunos preprocesadores son:

  • Sass (más utilizado).
  • Stylus (en desuso).
  • Less (en desuso).

Desventajas de CSS.

  • No hay forma de automatizar los estilos similares, se tienen que hacer manualmente.
  • No se pueden modificar fácilmente las guías de estilos (aunque ya existen variables CSS).
  • No se puede modularizar (dividir en partes) el código sin afectar el rendimiento del proyecto.
  • Es propenso a sobreescrituras, problemas de especificidad y cascada en proyectos grandes.

Ventajas de un preprocesador.

  • Estructurar los archivos en módulos, pero compilarlos a un único archivo CSS resultante.
  • Programar estilos automáticamente.
  • Almacenar las guías de estilo del proyecto en variable.
  • Hacer que proyectos grandes sean mantenibles entre equipos de trabajo.

Ventajas de usar Sass.

  • Usado por proyectos líderes.
  • Gran comunidad y soporte.
  • Sintaxis sólida y similar a CSS.
  • Se puede usar en cualquier entorno (no solo en Ruby) con libsass.

Sintaxis de Sass.

Existen 2 tipos de sintaxis en Sass:
  1. Sintaxis .Sass (no recomendada, en desuso)
  2. Sintaxis .Scss (recomendada)

Compilación.

Es necesario convertir el código Sass a CSS para que el navegador pueda entenderlo, para ello es necesario una herramienta para compilar Sass. Si el código no se compila no va a funcionar.

Herramientas desde el navegador.

  • Sass Meister (https://www.sassmeister.com/)
  • CodePen (https://codepen.io/)

Compiladores gráficos.

  • Prepros (recomendado) (https://prepros.io/)
  • CodeKit (https://codekitapp.com/)
  • Koala (http://koala-app.com/)
  • Scout (https://scout-app.io/)

Compilador por línea de comandos.

  • Ruby Sass (https://sass-lang.com/ruby-sass)
  • Node Sass (recomendable)
    1. Instalar node.js (https://nodejs.org/es/)
    2. Instalarlo en la terminal del editor de código con el comando npm install -g node-sass
    3. Probar si está instalado con el comando node-sass --version

Compilación de SCSS a CSS con Node.

  1. Abrir la terminal en el editor de código.

  2. Para compilar un archivo.scss a uno .css se escribir el comando node-sass archivo.scss archivo.css

    Para guardar la compilación automáticamente (sin generar el comando cada que se quiera compilar un archivo.scss) se escribe el comando node-sass --watch style.scss style.css

    Para guardar la compilación de un directorio a otro de forma automática se utiliza el comando node-sass --watch scss --output css (scss = carpeta de entrada; css = carpeta de salida). Esto hace que un archivo.scss de una carpeta guarde su compilación en un archivo.css (con el mismo nombre del .scss) de otra carpeta.

    Se puede utilizar un shorthand de --watch (-w) y --output (-o) de la siguiente forma: node-sass -w dir_entrada/scss -o dir_salida/css

Estructura del proyecto SASS.

Para crear un proyecto con buenas prácticas con SASS, se deben seguir los siguientes pasos:

  • Crear carpetas diferentes para almacenar los archivos de .scss y .css para mantener un orden en los archivos.
  • Separar los archivos .scss en módulos (subdirectorios).
  • Crear un archivo principal .scss que tendrá como función importar todos los módulos para obtener una sola salida, por lo que es preferible que no esté dentro de una subcarpeta.
  • En el html linkear solo el archivo.css que se obtendrá como salida de los módulos compilados.
Ͱ css
|
Ͱ sass
Ͱ- base
Ͱ-- _base.scss
|
Ͱ- components
Ͱ-- _banner.scss
Ͱ-- _buttons.scss
Ͱ-- _footer.scss
Ͱ-- _header.scss
Ͱ-- _icons.scss
Ͱ-- _menu.scss
|
Ͱ- config
Ͱ-- _mixins.scss
Ͱ-- _variables.scss
|
Ͱ- layouts
Ͱ-- _layout.scss
|
Ͱ- themes
Ͱ-- _theme.scss
|
Ͱ- style.scss

En el ejemplo anterior se colocarán los estilos de la siguiente manera:

- En la carpeta base, en el archivo _base.scss se colocarán los estilos básicos como *, *::before, *::after{}, color de las fuentes, tamaño de títulos, etc.
- En la carpeta components se colocan los archivos con los estilos de cada bloque del proyecto, como los estilos de los botones, de la cabecera, menú, etc.
- En la carpeta config se colocan los componentes que se configuran como los mixins o las variables de colores o medidas.
- En la carpeta layouts se colocan los archivos con estilos de layout como los grid y flex con sus columnas, filas y medidas de RWD.
- En la carpeta themes se colocan los archivos de los estilos decorativos o apariencia de la página que se visualiza.
- Por último, en el archivo styles.scss se importan todos los archivos para compilarlos y obtener un único archivo style.css.
 Se comienza por las variables y los mixins, después los estilos base y layout, después los componentes en el orden en que se utilizan en el proyecto (comenzando por el header y menú y finalizando con los componentes adicionales como botones e iconos) y por último se importan los temas del proyecto.

@import

Es importante que la salida de los archivos de tipo .scss de un proyecto sea un solo archivo de tipo .css. Para estructurar todos los archivos .scss en el proyecto, se utilizan importaciones.

menu.scss
.menu {
 background: pink;
}
styles.scss
@import "menu.scss";
// No es necesario colocar la extensión .scss pues solo se pueden importar archivos .scss
menu.css
.menu {
 background: pink;
}
styles.css
.menu {
 background: pink;
}

Lo ideal es crear subcarpetas en una carpeta llamada SASS en el proyecto para organizar los archivos dependiendo su contenido como variables o secciones del proyecto. Para evitar que las importaciones entre los archivos .scss creen archivos .css al compilarse, se agrega un guion bajo al comienzo del nombre del archivo .scss (convirtiéndolo en un partial) para indicarle al compilador que debe ignorar a ése archivo (no es necesario importar los archivos con el guion bajo), ejemplo:

_menu.scss
.menu {
 background: pink;
}
styles.scss
@import "menu.scss";
styles.css
.menu {
 background: pink;
}

Es posible especificar valores de las variables importadas, pero se debe tomar en cuenta la compilación en cascada.

_menu.scss
.menu {
 background: $color;
}
styles.scss
$color: pink;
@import "menu.scss";

.body{
 ...
}
styles.css
.menu {
 background: pink;
}

.body{
 ...
}

Lo más común y eficiente es declarar las variables del proyecto en un archivo llamado _config.scss e importarlo al inicio del archivo que generará la salida .css.
