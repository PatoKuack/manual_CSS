Sintaxis en SASS.

Comentarios.

Los comentarios son importantes para la documentación, tanto en SASS como en CSS, para mostrar comentarios en CSS desde SCSS se hace lo siguiente:

.scss
/*Comentarios visibles en la compilación.*/
// Comentarios invisibles en la compilación.
.css
/*Comentarios visibles en la compilación.*/

Variables.

Una variable almacena información que se pueda reutilizar.

En sass las variables se declaran con el signo monetario $ de la siguiente forma:

.scss
//Declaración de variable.
$size: 200px;
//Uso de la variable declarada.
.body {
 padding-top: $size;
}
//Además se pueden hacer operaciones con las variables.
.footer {
 //multiplicación:
 padding-top: $size * 2;
 //división:
 padding-top: $size / 2;
 //suma:
 padding-top: $size + 2;
 //resta:
 padding-top: $size - 2;
}
.css



.body {
 padding-top: 200px;
}

.footer {
 padding-top: 400px;
 padding-top: 100px;
 padding-top: 202px;
 padding-top: 198px;
}

Es posible modificar las unidades, eliminarlas o agregarlas.

.scss
$size: 5px;
.body{
 margin: ($size/1px)*1%;
}
.css

.body{
 margin: 5%;
}

Alcance.

También se pueden declarar variables dentro de algún selector o bloque para que solo funcionen dentro de dicho selector o bloque, no se podrá utilizar dicha variable fuera de ese bloque de código. Por ejemplo:

.scss
.body {
 $size: 20px;
 padding-top: $size;
}
.css
.body {
 padding-top: 20px;
}

!default

Para poder sobrescribir valores de una variable a lo largo del proyecto se utiliza !default, esto permite que una propiedad tenga un valor por defecto, permitiendo sobrescribir ese valor antes o después de su uso en dicha propiedad.

Es utilizado cuando se desea que personas ajenas al proyecto puedan cambiar valores de ciertas variables, como el color. Se usa de la siguiente manera:

.scss
$txt-color: purple !default;

.body{
 color: $txt-color;
}
$txt-color: green;
$txt-color: purple !default;

.body{
 color: $txt-color;
}
$txt-color: purple !default;
$txt-color: brown;

.body{
 color: $txt-color;
}
.css


.body{
 color: purple;
}



.body{
 color: green;
}



.body{
 color: brown;
}

Interpolación.

Una de las cosas que se pueden hacer en Sass es interpolar las variables. Interpolar variables es utilizar su valor para construir una expresión. Se hace de la siguiente manera:

.scss
$color: red;
$size: 100%;

// Calc requiere dos valores por lo que es necesario interpolar.
.#{$color}{
 height: calc($size - 50);
 height: calc(#{$size} - 50);
}
.css




.red{
 height: calc($size - 50);
 height: calc(100% - 50);
}

*Nota:
Las variables de SASS desaparecen al compilar el programa, así que sería recomendable utilizar las variables SASS solo para cálculos o acciones complejas y utilizar variables CSS con cálculos y acciones simples.

Anidamiento.

Anidamiento es poder meter selectores dentro de otros (no es buena idea usar muchas anidaciones porque el código se vuelve confuso). Por ejemplo:

.scss
.div {
 margin: 0;
 .p {
  margin: 0;
 }
 .ul {
  margin: 0;
 }
}
// El elemento ".p" y el ".ul" que está dentro de ".div" tendrá un valor 0 de margen.
.css
.div {
 margin: 0;
}
.div .p {
 margin: 0;
}
.div .ul {
 margin: 0;
}

Ejemplo de anidamiento con media queries:

.scss
.ul {
 display: block;
 @media screen and (min-width: 640px) {
  display: flex;
 }
}
.css
.ul {
 display: block;
}
@media screen and (min-width: 640px) {
 .ul {
  display: flex;
 }
}

Ejemplo de anidamiento con animaciones:

.scss
// Más fácil de visualizar.
.ul {
 animation: menu 1s;
 @keyframes menu {
  to {
   color: aliceblue;
  }
 }
}
.css

.ul {
 animation: menu 1s;
}
@keyframes menu {
 to {
  color: aliceblue;
 }
}

Otro tipo de sintaxis:

.scss
.menu {
 margin: 1rem;

 font: {
  family: arial;
 }
}
.css
.menu {
 margin: 1rem;

 font-family: arial;
}

Ampersand.

Otra forma de anidar es utilizando el ampersand (&) para indicarle a un pseudoelemento o a un selector que se aplicará al elemento que los contiene, ejemplo:

.scss
.ul {
 background-color: darkblue;

 &:hover{
  color: gold;
 }

 :hover{
  color: orange;
 }
}
.menu {
 margin: 1rem;

 &-item{
  margin: 5rem;
 }
}
.css
.ul {
 background-color: darkblue;
}

ul:hover{
 color: gold;
}

ul :hover{
 color: orange;
}
.menu {
 margin: 1rem;
}

.menu-item{
 margin: 5rem;
}

@extend

@extend permite crear placeholders e importarlos o extenderlos en selectores.

Los placeholders se usan para guardar bloques de estilos que podrán reutilizarse, ejemplo:

.scss
%button {
 display: inline-block;
 margin: 1em 0;
 padding: .5em 1.5em;
}

.button {
 @extend %button;
}
.css






.button {
 display: inline-block;
 margin: 1em 0;
 padding: .5em 1.5em;
}
.scss
%button {
 display: inline-block;
 margin: 1em 0;
 padding: .5em 1.5em;
}

.button {
 @extend %button;
}

.button-alert {
 @extend %button;
 // funciona igual con @extend .button;
 background-color: red;
}
.css






.button, .button-alert {
 display: inline-block;
 margin: 1em 0;
 padding: .5em 1.5em;
}

.button-alert {
 background-color: red;
}

También se pueden reutilizar bloques de código de otros selectores, ejemplo:

.scss
.button {
 display: inline-block;
 margin: 1em 0;
 padding: .5em 1.5em;
}

.button-alert {
 background-color: red;
 @extend .button;
 // a fuerza tiene que ser .button y no %button
}
.css
.button, .button-alert {
 display: inline-block;
 margin: 1em 0;
 padding: .5em 1.5em;
}

.button-alert {
 background-color: red;
}

Se tiene que tener bastante cuidado cuando se extienden los selectores. Es recomendable utilizar extend en selectores con solamente 1 nivel, ya que, si se desciende a subniveles esto puede causar complicaciones.

.scss
%button {
 display: inline-block;
 margin: 1em 0;
 padding: .5em 1.5em;
}

.div {
 display: block;

 .button {
  @extend %button;
 }

}
.css






.div .button {
 display: inline-block;
 margin: 1em 0;
 padding: .5em 1.5em;
}

.div {
 display: block;
}

@extend también se puede utilizar en pseudoclases, ejemplo:

.scss
.button:hover {
 transform: translateY(-5px);
}

.button:active {
 @extend .button:hover;
}
.css
.button:hover, .button:active {
 transform: translateY(-5px);
}
