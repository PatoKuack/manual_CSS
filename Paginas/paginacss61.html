Tailwind

Es un framework de CSS, es decir, un conjunto de herramientas para construir una interfaz web con CSS permitiendo agilizar el desarrollo.

Características.

  • Está representado mediante clases, al agregar clases a las etiquetas de HTML, se agregan los estilos.
  • Está orientado a utilidades por lo que lo vuelve personalizable.

Los frameworks basados en utilidades ( utility-first ) permiten tener una estructura limpia y ordenada en los estilos. Además, el desarrollo es rápido porque todo se trabaja dentro del mismo archivo.

Otros frameworks.

Bootstrap
  • Está basado en componentes.
  • Abstrae la creación de componentes.
  • Poco personalizable.
Material UI
  • Está basado en componentes pre-construidos.
  • Está basado en el tema de Material (como el sistema operativo de Android).
  • Es tan complejo que se vuelve un Design System.
Bulma
  • Es modular, es decir, permite importar solamente los archivos que se van a utilizar.
  • Es personalizable, ya que se le puede agregar cualquier estilo mediante la creación de variables en Sass.
  • Está basado en Flexbox.

Instalaciones.

Se tiene que inicializar NPM desde la terminal con el comando npm init y se agregan las especificaciones que se deseen o se puede dar enter hasta que acabe la inicialización de NPM.

Desde la documentación de tailwindcss se pueden ver varias guías para instalar Tailwind de diferentes formas como por CLI o por PostCSS y en este caso se explicará la instalación de PostCSS porque crea un archivo de configuración y facilita personalizar clases, además se puede hacer uso de todas las directivas de tailwind.

Como dice la documentación:

Primero se ejecuta en la terminal el comando (documentación - Install Tailwind vía npm):

npm install -D tailwindcss@latest postcss@latest autoprefixer@latest

Para configurar los Postcss plugins, se crean dos carpetas, una para almacenar las configuraciones y otra para compilarlas, en este caso se crearán en la raíz del proyecto de la siguiente forma:

  |- public
  | |- index.html
  | |- css
  |   |- tailwind.css
  |
  |- src
  | |- css
  |   |- tailwind.css

En el archivo src/css/tailwind.css se agrega lo siguiente (documentación - including Tailwind in your CSS):

@tailwind base;
@tailwind components;
@tailwind utilities;

En el archivo html se vincula el estilo (en este caso) public/css/tailwind.css

Después se crea el archivo de configuración (tailwind.config.js) con el que se podrá personalizar colores o tiopgrafías, ejecutando el comando (documentación - Create your configuration file):

npx tailwindcss init

Se obtendrá el archivo de configuración con un código simple en el que se agregará la ruta del html y otros datos que se especifican en la documentación:

module.exports = {
 content: ["./public/index.html", "./src/**/*.{html,js}"],
 theme: {
   extend: {},
 },
 variants: {},
 plugins: [],
}

Antes de utilizar tailwind, se indica que el archivo tailwind.css que proviene del directorio src/css haga un build en el archivo public/css/tailwind.css, para esto se ejecuta el comando (documentación - Using a custom CSS file) (en este caso se utilizan las siguientes rutas):

npx tailwindcss -i ./src/css/tailwind.css -o ./public/css/tailwind.css -w

La instrucción -w mantiene compilando los cambios sin repetir el código a cada modificación, pero se tiene que abrir otra terminal ya que este comando bloquea la terminal mientras este en uso. Para parar el comando se presiona control+C desde la terminal.

Ya que se puede utilizar tailwind, se prueba agregando algún estilo como un div class= "w-64 h-64 bg-red-200"

Facilidades.

Extensión:

Si se utiliza Visual Studio Code, se puede instalar la extensión Tailwind CSS IntelliSense para visualizar colores o recibir sugerencias de comandos.

npm run.

Para no digitar todo el código de compilación, se utiliza un script en el documento Package.json

En el bloque de "scripts" se encuentra por default el script "test", al final de su línea de código se coloca una coma y se agrega la siguiente línea:

"tw:build": "tailwindcss -i ./dir_entrada/tailwind.css -o ./dir_salida/tailwind.css --watch"

Y de esta manera solo se tiene que colocar en la terminal el comando: npm run tw:build (el nombre del objeto del script puede ser el que se desee)

Directivas.

Una directiva es una instrucción que inserta código al archivo final de CSS. Existen muchas directivas que se especifican en la documentación (Functions & Directives).

@tailwind base;
Inicializa todos los elementos de nuestro HTML sin estilo.
@tailwind components;
Nos permite acceder a todas las clases de tailwind.
@tailwind utilities;
Agrega todas las utilidades al proyecto y nos permite acceder a ellas.
@tailwind screen;
Modifica breakpoints a la preferencia de uno.
@layer
Especifica qué bloque de código pertenece a qué estilo.
@variants
Agrega estilos responsivos orientados a manipulación de eventos como :hover, :focus o :active.
@responsive
Genera diferentes estilos responsivos en las clases.
theme()
Accede a los valores de configuración iniciales de tailwind

Breackpoints.

El breackpoint es el cambio de layout de la pantalla.

Las medidas dentro de tailwind son:

sm max-width: 640px;
md max-width: 768px;
lg max-width: 1024px;
xl max-width: 1280px;
2xl max-width: 1536px;

Para especificar los estilos para versiones web del proyecto se utiliza el breakpoint especificándolo antes de agregar las clases, ejemplo: lg:w-full lg:h-full

Dark-mode.

Para agregar el modo oscuro en un proyecto, se agrega el siguiente bloque en el archivo tailwind.config.js:

content: [ ... ],
module.exports = {
 darkMode: 'class',
},
 theme: { ... },

Después se agregan las clases que se deseen en el modo oscuro al final de la clase de la siguiente manera: class="bg-white dark:bg-black text-white" ...

El modo oscuro se aplica de diferentes formas, por ejemplo, si la etiqueta HTML tiene la clase "dark" o si se indica el tema mediante la memoria del navegador (localstorage), para colocar esta última se tiene que agregar un documento javascript con el siguiente código:

Aspect ratio

Aspect ratio es la relación entre el ancho y el alto de la imagen o video. En muchas ocasiones cuando se modifican las dimensiones de elementos o imágenes, se pueden llegar a tener problemas con el aspect ratio, pero con este plugin se resuelve el problema integrando la clase 'aspect' que permite establecer un aspect ratio de manera fija para los elementos.

Personalización.

Para agregar personalizaciones en Tailwind, se deben agregar las siguientes configuraciones en el archivo Tailwind.config.js:

Agregar imágenes de fondo

Se tienen que tener las imágenes ubicadas dentro del proyecto.

theme: {
 extend: {
  backgroundImage: {
   'imageName': "url(directorio/de/la/imagen)",
  },
 },
},

Agregar colores de fondo.

Se incorporan los colores nativos de tailwind con ...theme

theme: {
 extend: {
  backgroundColor: theme =>({
   ...theme('colors'),
   'colorName': 'hexadecimalColor',
  }),
 },
},

Ahora se podrá utilizar en las clases, por ejemplo: bg-colorName

Agregar colores de texto.

Aunque se utilice el mismo color en un texto y en un fondo, se tiene que colocar por separado el color en backgroundColor y en textColor.

theme: {
 extend: {
  textColor: {
   'colorName': 'hexadecimalColor',
  },
 },
},

Ahora se podrá utilizar en las clases, por ejemplo: text-colorName

Agregar colores de outline.

theme: {
 extend: {
  ringColor: {
   'colorName': 'hexadecimalColor',
  },
 },
},

Ahora se podrá utilizar en las clases, por ejemplo: focus:ring-colorName

Agregar fuentes de texto.

Se agregan las fuentes a la sección de head del archivo HTML como si se utilizaran normalmente sin tailwind.

theme: {
 extend: {
  fontFamily: {
   fontName: ['font', 'subfont'],
  },
 },
},

Ahora se podrá utilizar en las clases, por ejemplo: text-fontName

Después de cada modificación se compila el proyecto (desde la consola: tw:build).

Ahora no nada más se puede colocar un color con class="bg-red-200", sino también con class="bg-colorName". También se puede colocar la fuente personalizada con class="font-fontName".

Extracción de componentes.

La extracción de componentes sirve para no escribir el mismo código en componentes que se utilizarán más de una vez y poder mantener el principio de utility-first. La extracción de componentes trabaja de la mano con la directiva @apply.

En la documentación (Reusing styles) se puede saber cómo utilizar la extracción de componentes.

En el archivo src/css/tailwind.css se crean clases con el nombre deseado y con la primera letra en mayúscula, dentro de la clase se introducen todos los estilos de Tailwind que se estarán reutilizando haciendo el uso de la directiva @apply seguido de las clases de Tailwind, ejemplo:

@tailwind base;
@tailwind components;
@tailwind utilities;
.clase-name {
 @apply w-48 h-64 shadow-md rounded-lg
}

Se compila el proyecto y después se podrá utilizar el nombre de la nueva clase en el archivo HTML.

Tailwind no está diseñado para trabajar con muchas clases usando @apply, por lo que se debe identificar y limitar la cantidad de clases que se crearán.

Purge.

El Purge o Purga, limpia el proyecto removiendo archivos que no se utilizan y mimifica el código. Para purgar se inserta el siguiente código en el archivo de configuración tailwind.config.js:

module.exports = {
 plugins: {
  tailwindcss: {},
  autoprefixer: {},
  ...(process.env.NODE_ENV === 'production' ? { cssnano: {} } : {})
  }
}
