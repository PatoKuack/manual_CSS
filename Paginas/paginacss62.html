Instalación

Se tiene que inicializar NPM desde la terminal con el comando npm init y se agregan las especificaciones que se deseen o se puede dar enter hasta que acabe la inicialización de NPM.

Desde la documentación de tailwindcss se pueden ver varias guías para instalar Tailwind de diferentes formas como por CLI o por PostCSS y en este caso se explicará la instalación de PostCSS porque crea un archivo de configuración y facilita personalizar clases, además se puede hacer uso de todas las directivas de tailwind.

Como dice la documentación:

Primero se ejecuta en la terminal el comando (documentación - Install Tailwind vía npm):

npm install -D tailwindcss@latest postcss@latest autoprefixer@latest

Para configurar los Postcss plugins, se crean dos carpetas, una para almacenar las configuraciones y otra para compilarlas, en este caso se crearán en la raíz del proyecto de la siguiente forma:

|-public
| |-index.html
| |-css
|   |-tailwind.css
|
|-src
| |-css
|   |-tailwind.css

En el archivo src/css/tailwind.css se agrega lo siguiente (documentación - including Tailwind in your CSS):

@tailwind base;
@tailwind components;
@tailwind utilities;

En el archivo html se vincula el estilo (en este caso) public/css/tailwind.css

Después se crea el archivo de configuración (tailwind.config.cjs) con el que se podrá personalizar colores o tiopgrafías, ejecutando el comando (documentación -> Create your configuration file):

npx tailwindcss init

Se obtendrá el archivo de configuración con un código simple en el que se agregará la ruta del html (./public/index.html) o de varios html (./public/*.html) y otros datos que se especifican en la documentación:

module.exports = {
 content: ["./public/index.html", "./src/**/*.{html,js}"],
 theme: {
   extend: {},
 },
 variants: {},
 plugins: [],
}

Antes de utilizar tailwind, se indica que el archivo tailwind.css que proviene del directorio src/css haga un build en el archivo public/css/tailwind.css, para esto se ejecuta el comando (documentación - Using a custom CSS file) (en este caso se utilizan las siguientes rutas):

npx tailwindcss -i ./src/css/tailwind.css -o ./public/css/tailwind.css -w

La instrucción -w mantiene compilando los cambios sin repetir el código a cada modificación, pero se tiene que abrir otra terminal ya que este comando bloquea la terminal mientras este en uso. Para parar el comando se presiona control+C desde la terminal.

Ya que se puede utilizar tailwind, se prueba agregando algún estilo como un div class= "w-64 h-64 bg-red-200"

Resumen:

Desde la consola: npm init

Desde la consola: npm install -D tailwindcss@latest postcss@latest autoprefixer@latest

Desde la consola: npx tailwindcss init

Se agrega la ruta del html en tailwind.config.js: "./public/index.html", "./src/**/*.{html,js}"

Se crea el directorio de entrada y salida para la compilación del framework. En el de entrada se agregan las directivas: @tailwind base; @tailwind components; @tailwind utilities;

Se vincula el archivo public/src/tailwind.css con el archivo html

Desde la consola: npx tailwindcss -i ./dir_entrada/tailwind.css -o ./dir_salida/tailwind.css --watch

Facilidades.

Extensión:

Si se utiliza Visual Studio Code, se puede instalar la extensión Tailwind CSS IntelliSense para visualizar colores o recibir sugerencias de comandos.

npm run:

Para no digitar todo el código de compilación, se utiliza un script en el documento Package.json

En el bloque de "scripts" se encuentra por default el script "test", al final de su línea de código se coloca una coma y se agrega la siguiente línea:

"tw:build": "tailwindcss -i ./dir_entrada/tailwind.css -o ./dir_salida/tailwind.css --watch"

Y de esta manera solo se tiene que colocar en la terminal el comando: npm run tw:build (el nombre del objeto del script puede ser el que se desee)
